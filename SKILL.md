# 对标账号深度拆解

通过分析对标账号的文章内容，运用"测试思维"进行深度拆解，帮助用户从"找对标"过渡到"找定位"。

对标的三个核心角度：
1. **找爆款选题** — 什么选题用户喜欢？什么选题数据好？什么选题适合我？
2. **找内容结构** — 标题怎么写？开头怎么写？内容怎么展开？为什么这样的结构能吸引人？
3. **找变现路径** — 这个账号卖什么？用户为什么买？我能不能复制？

## 核心理念

- **定位是长出来的**：好的定位不是想出来的，而是在寻找对标、打磨内容的土壤里长出来的
- **测试思维**：不纠结空泛的定位，通过看市场、看内容、看变现来测试方向
- **定位公式**：定位 = Who（谁）+ Where（什么场景）+ What（解决什么问题）+ How（怎么解决）

## 启动流程

技能触发后，**一次性**通过 AskUserQuestion 收集以下 3 个信息，之后全程零交互直到报告生成：

| # | 问题 | 类型 | 选项 |
|---|------|------|------|
| 1 | 对标账号名称是什么？ | 自由输入 | — |
| 2 | 选择分析模式 | 单选 | 快速拆解（覆盖主流平台，建议 10-15 篇）/ 深度拆解（尽量搜齐，30+ 篇） |
| 3 | 想从拆解中获得什么？ | 单选 | 选题灵感 / 内容结构模仿 / 变现路径参考 / 全面拆解 / 其他（自由输入） |

问题 3 的作用：选择的方面在最终「行动建议」章节中重点展开，其他方面正常输出但不展开。

## 搜索策略

### 平台优先级

按实际可搜索能力排序：
1. **X (Twitter)** — 用 `opentwitter-link` skill 通过 API 获取用户信息和推文（需 `$TWITTER_TOKEN`）
2. **知乎** — 搜索用户主页/专栏文章
3. **小红书** — 搜索用户笔记（CDP 浏览器）
4. **B站专栏** — 搜索用户专栏文章（只抓图文，视频只记录标题）
5. **虎嗅/头条/腾讯新闻/搜狐** — 搜索转载内容
6. **其他** — 微博、人人都是产品经理等

文章类内容优先，视频平台只记录标题和简介。

### 公众号内容获取规则

公众号文章无法通过搜索引擎直接搜索全文。按以下优先级获取：
1. **用户提供**：用户在启动时或过程中提供的公众号文章链接/本地文件/粘贴文本，直接使用
2. **搜索转载源**：其他平台的搜索结果中如果出现公众号文章的转载版本（头条、腾讯新闻、搜狐、虎嗅等），通过转载源获取内容并标注"转载自公众号"
3. **不强求**：如果搜索中未出现公众号转载内容，不单独为公众号渠道花费搜索精力，以其他平台数据为主，在报告中说明

### 搜索流程（全自动，零交互）

**第一步：确认主阵地**
用账号名搜索，判断该账号的活跃平台和主阵地。搜完直接汇报：
> "初步判断「XX」主阵地是 X 和公众号，优先从这两个平台抓取。"

**第二步：逐平台搜索 + 实时汇报**
按优先级逐平台搜索，每完成一个平台汇报一行进度：
> "✅ 公众号：找到 12 篇图文
> ✅ X：找到 20 条 thread（碎片化）
> → 正在搜索知乎..."

用户随时可发消息干预（如"够了"、"跳过 XX 平台"、"去搜 XX 平台"）。

**第三步：达标即停**
- 快速模式：建议 10-15 篇即可停止搜索
- 深度模式：主流平台全部扫完，尽量搜齐 30+ 篇
- 达标后直接进入分析，不询问确认

**用户提供材料**
如果用户在启动时或过程中提供了文章链接、本地文件或粘贴文本，直接使用：
- 微信文章链接：用 `web-access` skill 通过 CDP 浏览器抓取
- 本地文件（markdown/txt）：直接读取
- 粘贴文本：直接使用
用户提供的材料与搜索材料合并分析。如果该账号全网内容极少（小众账号），以已有材料为主，在报告中说明信息来源有限。

## 分析工作流

合并所有材料后，按以下 6 章分析。**每条结论必须基于具体文章证据，不要凭空推测。**

### 账号概览（简要，5-10 行）

- 赛道：该账号属于什么赛道？
- 内容风格：文章整体风格（干货清单、故事叙述、观点输出、工具测评等）
- 信息来源汇总表

### 第 1 章：爆款选题分析

- **选题分布**：该账号的选题覆盖哪些方向？各方向占比如何？
- **爆款归因**：引用具体文章标题分析为什么数据好（标题吸引力、选题精准度、情绪价值、实用性、热点借势）
- **选题规律**：什么类型的选题反复出现？什么选题数据稳定？什么选题偶尔爆发？
- **选题时机**：是否紧跟热点？发布节奏如何？

### 第 2 章：内容结构拆解

- **标题写法**：标题有什么共同模式？（数字型、疑问型、对比型、利益点前置等）
- **开头 Hook**：开头怎么抓住注意力？（一句话价值承诺、痛点共鸣、反常识、故事切入等）
- **正文展开**：内容怎么组织？（总分总、步骤式、并列式、对比式、故事线）
- **格式特征**：是否有标志性格式？（emoji 编号、加粗金句、表格对比、代码块等）
- **结尾收束**：结尾怎么收？（总结金句、行动号召、引导互动、留悬念）

### 第 3 章：用户画像与定位

- **用户画像**：推断读者的年龄、职业、焦虑点、阅读场景
- **核心痛点**：用户在阅读前内心最焦虑的问题是什么？
- **核心抓手**：该账号真正打动用户的核心点（身份反差、公开成长过程、信息差/认知差、代表某类人群发声）
- **不可替代的差异化点**：为什么读者选他而不是别人？
- **定位公式**：

| 维度 | 输出 |
|------|------|
| **Who（谁）** | 具体的人群标签 |
| **Where（场景）** | 具体的场景描述 |
| **What（问题）** | 一句话概括核心痛点 |
| **How（方式）** | 通过什么方式解决 |

> 定位公式：[Who] 在 [Where] 场景下，通过 [How] 解决 [What] 问题

### 第 4 章：变现路径分析

- **变现方式**：该账号卖什么？（付费专栏、知识星球、咨询服务、自有产品、广告、课程等）
- **用户为什么买单**：用户付费的核心驱动力是什么？（节省时间、提升认知、获得工具、情绪价值、社交资本）
- **商业模式**：这个赛道的常见变现模式是什么？该账号的变现属于哪种？
- **可复制性**：这个变现模式能不能复制？门槛在哪里？

### 第 5 章：行动建议

**选题方向**（从拆解文章提炼 3-5 个方向）：
- 每个方向附具体文章标题作为证据
- 标注该方向的预计数据表现和竞争程度

**结构模板**：
- 提炼该账号可复用的文章结构模板
- 标注适用于什么类型的选题

**差异化切入点**：
- 结合用户自身情况（问题 3 的选择）寻找定位空间
- 寻找"别人不想做/挤不进去"的反面机会
- 如果用户提供了自身背景，给出针对性的定位建议

## 保存逻辑

报告生成完成后，询问用户保存位置（AskUserQuestion），文件名格式：`对标拆解-[账号名]-YYYY-MM-DD.md`

## 注意事项

- 每条结论都应能回溯到具体文章证据
- 行动建议要具体可执行，不要给"坚持输出"这种正确但无用的建议
- 如果信息来源有限，在报告中明确说明，避免过度推断
- 分析过程中实时汇报搜索进度，但不要主动询问确认（用户可随时发消息干预）
